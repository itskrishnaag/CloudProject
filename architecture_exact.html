Key architectural decisions (Phase 3):
  • Consumer group split: edtech-monitoring-group (Spark → Redis, latency-sensitive) and edtech-storage-group (k8s pods → S3+DynamoDB, throughput-sensitive). Both receive ALL events independently.
  • STORAGE_ENABLED=false on Spark: Spark no longer writes to S3 or DynamoDB. All durable storage is handled by storage_worker.py pods. Prevents duplicate writes.
  • Two-tier scaling: Tier 1 = KEDA pods (15s response, ~40s 1→10), Tier 2 = ASG EC2 nodes (~3–4 min, only triggers when EC2-2 CPU > 70% = pods pending). Scale-to-zero at baseline = zero idle EC2 cost.
  • S3 file naming: Pod files have UUID suffix ({epoch}_{uuid8}.parquet) to distinguish from legacy Spark files. Hive-partitioned by year/month/day/hour.
  • Race condition fix: session_timeout_ms=45000 prevents pods being evicted during Kafka group rebalance when KEDA rapidly scales 1→10.
  • CloudWatch watches EC2-2 InstanceId (not ASG avg CPU) — ASG starts at 0 instances, has no CPU data. EC2-2 is always running and its CPU reflects pod scheduling pressure.
